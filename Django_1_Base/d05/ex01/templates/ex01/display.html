Ex01: Display process of a static page

Description: In this page, describe the process causing the display of a static web page from a simple template going through a view, from the request to the answer.

Introduction

Displaying a static web page in Django involves a series of steps that start from the user's request and end with the server's response. This process includes routing the request, processing it in a view, rendering a template, and returning the response to the client.

Step-by-Step Process

1. User Request

The process begins when a user makes a request by entering a URL in their browser. This request is sent to the Django server.

2. URL Routing

Django uses URLconf (URL configuration) to map the requested URL to a specific view. This is done in the urls.py file. For example:


# urls.py
from django.urls import path
from . import views

urlpatterns = [
    path('display_process/', views.display_process, name='display_process'),
]

3. View Function

The view function associated with the URL processes the request. It retrieves any necessary data, processes it, and prepares it for the template. For a static page, this might simply involve rendering the template without any data processing:


# views.py
from django.shortcuts import render

def display_process(request):
    return render(request, 'display_process.html')

4. Template Rendering

The view function calls the render function, passing the request object and the template name. Django's template engine processes the template and combines it with any context data provided.

5. HTTP Response

Once the template is rendered, Django creates an HttpResponse object containing the rendered HTML. This response is sent back to the user's browser.

Example

For the URL /display_process/, the following steps occur:

  • The user requests /display_process/.
  • Django's URL dispatcher maps this URL to the display_process view function.
  • The view function renders the display_process.html template.
  • The rendered HTML is returned to the user's browser, which displays the static page.

Conclusion

This process highlights the simplicity and efficiency of Django in handling web requests and serving static content. By following these steps, Django ensures a smooth transition from request to response, providing a seamless user experience.
